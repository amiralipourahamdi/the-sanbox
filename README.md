# The Heysam Sandbox 🏗️

Welcome to the Sandbox. This repository is our shared training ground. It contains no production code, connects to no live databases, and impacts zero customers.

This is where we experiment, break things, and—most importantly—build our shared knowledge library.

## Mission 1: The PR Check-In

Before you touch any real product repositories, you must prove you understand our Git workflow. Your first task is to add yourself to this repository.

1. Clone this repository to your local machine.
2. Create a new branch: `git checkout -b intro/your-first-name`.
3. Scroll down to the **Active Roster** section at the bottom of this file.
4. Add a new bullet point with your Name, your primary technical focus, and a one-sentence introduction.
5. Commit the change, push your branch, and open a Pull Request against `main`.
6. Ensure your PR description is clear. Wait for a review and approval.

---

## Mission 2: Learn by Building (The Mini-Project Strategy)

At Heysam, we do not code blind, and we do not passively watch tutorials. If you encounter a complex concept, architecture, or tool that you do not fully understand, you must execute a "Mini-Project."

A mini-project is an isolated, throwaway app or script designed to teach you exactly how a mechanism works before you try to integrate it into our real products.

### How to Submit a Mini-Project

Whenever you complete a learning project, you will push it to this repository so the rest of the team can learn from it.

1. **Create a Folder:** Place your project inside the `/learning` directory. Name it clearly using the format: `/learning/[your-name]-[concept]` (e.g., `/learning/ali-docker-nginx-proxy`).
2. **Keep it Clean:** Do not commit `node_modules`, virtual environments, or heavy dependencies. Use `.gitignore`.
3. **Write a Mini-README:** Every project folder must contain its own `README.md` that answers three questions:
   - What concept were you trying to learn?
   - How does this code demonstrate it?
   - What was your biggest "Aha!" moment or blocker?
4. **Share It:** Once merged into `main`, drop a link to your folder in the `#engineering` Discord channel so the team can review it.
